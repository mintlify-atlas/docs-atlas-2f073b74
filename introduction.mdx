## What is CrudPark?

CrudPark is a modern parking management API built with ASP.NET Core 9.0 and PostgreSQL. It provides a complete solution for managing parking operations, including customer memberships, parking rates, real-time occupancy tracking, and financial reporting.

The API is designed using Clean Architecture principles, ensuring maintainability, testability, and separation of concerns across all layers of the application.

## Core Capabilities

- **Operator Authentication**: Secure operator management with password hashing and active status tracking
- **Customer Profiles**: Maintain customer information with support for multiple memberships per customer
- **Stay Tracking**: Monitor vehicle entry/exit with automatic payment calculation
- **Payment Processing**: Record and track all parking payments with detailed transaction history
- **Email Notifications**: Automated alerts for expiring memberships using background services
- **CORS Support**: Pre-configured for integration with frontend applications
- **Swagger Documentation**: Interactive API documentation available at `/swagger`

## Who Should Use This API?

CrudPark is ideal for:

- **Parking Facility Operators**: Organizations managing parking lots or garages that need automated tracking and billing
- **Property Management Companies**: Businesses overseeing parking for residential or commercial properties
- **Software Developers**: Teams building parking management applications who need a robust backend API
- **System Integrators**: Developers looking to integrate parking management into larger facility management systems

## Technology Stack

- **Framework**: ASP.NET Core 9.0
- **Database**: PostgreSQL with Entity Framework Core
- **Architecture**: Clean Architecture with distinct layers (API, Application, Core, Infrastructure)
- **Email Service**: SMTP integration for notifications
- **Containerization**: Docker support with Docker Compose
