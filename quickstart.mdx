## Prerequisites

Before you begin, ensure you have the following installed:

- **.NET 9 SDK** - [Download here](https://dotnet.microsoft.com/download/dotnet/9.0)
- **PostgreSQL** - Version 12 or higher
- **Docker** (optional) - For containerized deployment
- **Git** - To clone the repository

## Installation

<Steps>
  <Step title="Clone the Repository">
    Clone the CrudPark repository to your local machine:
    
    ```bash
    git clone https://github.com/yourusername/crudpark.git
    cd crudpark
    ```
  </Step>

  <Step title="Configure Database Connection">
    Update the connection string in `CrudPark.Api/appsettings.json` to point to your PostgreSQL instance:
    
    ```json CrudPark.Api/appsettings.json
    {
      "ConnectionStrings": {
        "DefaultConnection": "Host=localhost;Database=CrudPark;Username=postgres;Password=yourpassword;Port=5432"
      },
      "AppSettings": {
        "TotalParkingSpaces": 150
      }
    }
    ```
    
    <Note>
      The `TotalParkingSpaces` setting is used to calculate occupation percentages in dashboard metrics.
    </Note>
  </Step>

  <Step title="Apply Database Migrations">
    Create the database schema using Entity Framework Core migrations:
    
    <CodeGroup>
    ```bash Terminal
    cd CrudPark.Api
    dotnet ef database update
    ```
    
    ```bash Using dotnet-ef tool (if not installed)
    dotnet tool install --global dotnet-ef
    dotnet ef database update --project CrudPark.Api
    ```
    </CodeGroup>
    
    This will create all necessary tables including `operators`, `customers`, `memberships`, `rates`, `payments`, and `stays`.
  </Step>

  <Step title="Configure Email Settings (Optional)">
    If you want to enable membership expiration notifications, configure SMTP settings in `appsettings.json`:
    
    ```json CrudPark.Api/appsettings.json
    {
      "SmtpSettings": {
        "Host": "smtp.gmail.com",
        "Port": 587,
        "Username": "your-email@gmail.com",
        "Password": "your-app-password",
        "FromEmail": "noreply@crudpark.com",
        "FromName": "CrudPark System"
      }
    }
    ```
    
    <Warning>
      Never commit sensitive credentials to version control. Use environment variables or secret management tools in production.
    </Warning>
  </Step>

  <Step title="Run the Application">
    Start the API server:
    
    <CodeGroup>
    ```bash Development Mode
    dotnet run --project CrudPark.Api
    ```
    
    ```bash Production Build
    dotnet build -c Release
    dotnet run --project CrudPark.Api --configuration Release
    ```
    
    ```bash Using Docker
    docker build -t crudpark-api .
    docker run -p 8080:8080 crudpark-api
    ```
    </CodeGroup>
    
    The API will start on `https://localhost:5001` (or `http://localhost:5000`).
  </Step>

  <Step title="Verify Installation">
    Test the API by making your first request to the dashboard metrics endpoint:
    
    ```bash cURL
    curl -X GET "https://localhost:5001/api/dashboard/metrics" -k
    ```
    
    ```http HTTP Request
    GET /api/dashboard/metrics HTTP/1.1
    Host: localhost:5001
    ```
    
    Expected response:
    
    ```json Response
    {
      "vehiclesCurrentlyInside": 0,
      "occupationPercentage": 0.0,
      "incomeToday": 0.0,
      "membershipsActive": 0,
      "membershipsExpiringSoon": 0,
      "membershipsExpired": 0
    }
    ```
  </Step>
</Steps>

## Access Swagger Documentation

Once the API is running, you can explore all endpoints interactively:

1. Navigate to `https://localhost:5001/swagger` in your browser
2. The Swagger UI displays all available endpoints organized by controller
3. Test endpoints directly from the interface

<Info>
  Swagger is enabled in all environments by default. See `CrudPark.Api/Program.cs:70-71` for configuration.
</Info>

## Docker Compose Deployment

For a simplified deployment, use the provided `compose.yaml`:

```bash
docker-compose up --build
```

This will:
- Build the CrudPark API image
- Start the container on port 3000
- Set the environment to Production

## CORS Configuration

The API is pre-configured with CORS support for:
- `https://crudpark.netlify.app` (production frontend)
- `http://localhost:5173` (local development frontend)

See `CrudPark.Api/Program.cs:20-31` to modify allowed origins.
